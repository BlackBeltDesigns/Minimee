Minimee Syntax

There are now 3 "types" of tags available:

  1. "Traditional" tag pairs, which wrap <link> or <script> tags, parses them to find assets, and creates cache
  2. An "API" tag interface which operates entirely off of parameter values. By default it recieves an asset url, and returns a URL to its cache
  3. "Display" tags which are used for assets which have been queued for later output

"Traditional"

{exp:minimee:css} {/exp:minimee:css} {exp:minimee:js:embed} {/exp:minimee:js:embed}

"API"

{exp:minimee css="{stylesheet=global_embeds/site_css}"} {exp:minimee css="{stylesheet=global_embeds/site_css}" queue="head"} {exp:minimee:display:tag css="head"}

Cache a single file and return URL:

{exp:minimee css="{stylesheet=global_embeds/site_css}"} {exp:minimee css="{stylesheet=global_embeds/site_css},{stylesheet='global_embeds/css_screen-ie7'}"} {exp:minimee css="{stylesheet=global_embeds/site_css}|{stylesheet='global_embeds/css_screen-ie7'}" combine="no" files_delimiter="|"}

Cache a single file and return contents:

{exp:minimee:css:contents} {/exp:minimee:css:contents}
